# zsh

## Intro

zsh is a type of shell among many others and the following will be a guide on organizing what goes into each of the zsh configuration files.

> **NOTE:**
>
> There are other components one of which is `~/.zlogin`, which will not be covered here because I don’t think it’s as important.

------------------------------------------------------------------------

## .zshrc

This is the configuration file for the zsh shell.

Use it to set aliases and a custom prompt for the terminal window.

Most people also use this one script to also set the `PATH` but let’s do that in the `~/.zprofile`.

I personally use this to set CLI tools and to customize my prompts.

------------------------------------------------------------------------

## .zprofile

MacOS uses this to set up the shell for any new terminal window.

Homebrew recommends setting the `PATH` variable here. There’s a reason it should be set here and not the `~/.zshenv` file: macOS runs a utility `path_helper` (from `/etc/zprofile`) that sets the `PATH` order before `~/.zprofile` is loaded.

------------------------------------------------------------------------

## .zshenv

This is loaded universally for all types of shell sessions. MacOS overrides this for `PATH` settings for interactive shells.

This could be used to configure the shell for automated processes like cron jobs. However, it is not used as most variables are best to stay within the project directory with a `.env`.
